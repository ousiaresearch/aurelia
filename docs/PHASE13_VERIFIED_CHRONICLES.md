# Aurelia Phase 13 — Verified Chronicles

Phase 13 turns verified causal histories into readable civilization chronicles. The rule is simple: **no invented history**.

A chronicle is not allowed to outrun the run artifact. It must begin as a deterministic, auditable card with source paths and evidence. LLM prose can come later, but it must sit on top of the verified card rather than replacing it.

## Chronicle contract

A verified chronicle card must include:

- run ID
- world ID
- year
- source summary path
- source world DB path
- population, birth, death, and faction metrics when available
- event evidence from `causal_events` or `causal_summary.json`
- provenance status: `verified` or `partial`
- a committed provenance manifest when the chronicle is published under `docs/reports/`

The first implementation target is `scripts/render_verified_chronicles.py`, which emits deterministic Markdown and a compact JSON provenance manifest from a run directory.

## Why deterministic first

LLM prose can only be as trustworthy as the evidence scaffold beneath it. The first renderer is deterministic so every statement is auditable:

- If the source DB exists and the summary exists, provenance is `verified`.
- If evidence is missing, the card must be marked `partial` rather than padded with narrative.
- The renderer exposes source paths in the Markdown so an operator can trace the card back to the local run artifact.
- The renderer can also write a committed `.provenance.json` manifest that embeds run metadata, card metrics, evidence event types, and source file names without host-local absolute paths. This keeps published reports auditable even after `/tmp` run directories are gone.

## Operator command

```bash
PYTHONPATH=. python3 scripts/render_verified_chronicles.py \
  --run-dir /path/to/aurelia-run \
  --output docs/reports/phase13-verified-chronicles.md \
  --manifest-output docs/reports/phase13-verified-chronicles.provenance.json \
  --llm-output docs/reports/phase13-llm-chronicles.md \
  --prompt-output docs/reports/phase13-llm-prompts.jsonl
```

## LLM layer rule

The LLM layer may generate prose only after deterministic cards exist. Generated prose must cite or carry the card evidence, not replace it.

`scripts/render_verified_chronicles.py` now emits two LLM-layer artifacts:

- `docs/reports/phase13-llm-prompts.jsonl` — one evidence-locked prompt packet per verified card. These packets include portable card metadata, metrics, evidence event types, and source file names. They do not include host-local paths.
- `docs/reports/phase13-llm-chronicles.md` — offline grounded draft sections that preserve each card's evidence ledger and pass the evidence validator. These are scaffolds for a future model pass, not unconstrained fiction.

The prompt contract says: **do not invent named events, people, factions, outcomes, dates, places, or causal relationships.** If a model omits required evidence terms or card metadata, the draft fails validation and must not be published as a chronicle.

The safe progression is:

1. render verified cards from run artifacts;
2. inspect provenance and evidence lines;
3. emit prompt packets from those cards;
4. generate or revise prose from the prompt packets;
5. validate that prose still carries required evidence;
6. publish prose only alongside the deterministic evidence card and manifest.

## Publication standard

A Phase 13 chronicle artifact is publishable only if:

- it names the source run;
- it names the world and year;
- it includes source summary and DB paths;
- it links to a committed provenance manifest when published;
- it exposes event evidence or explicitly marks the card partial;
- any LLM-generated prose passes the evidence validator before publication;
- it does not introduce named events, people, factions, or outcomes absent from the source evidence.

This keeps Aurelia's narrative layer tied to its core claim: history in this system is generated by causal state, not invented after the fact.
